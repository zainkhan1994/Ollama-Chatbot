Request Lifecycle

How a Chat Request Flows

From HTTP call to streamed LLM response — tracing one request through the full stack.

1
HTTP Request In
Client POSTs to POST /chat. Spring MVC routes it to the controller, extracting the user message and session ID.
2
Advisors Applied
Spring AI Advisors intercept the request — loading chat memory, injecting system prompts, applying guardrails, or augmenting with RAG context.
3
Prompt Built
The ChatClient assembles the final prompt: system message + retrieved context + conversation history + user input.
4
Ollama Inference
Spring AI calls Ollama's local HTTP API (or OpenAI/Mistral for cloud). The model streams tokens back in real time.
5
Response + Memory
The AI reply is streamed to the client. Memory advisors persist the exchange to chat history so future turns have context.
Java · Spring AI 
@RestController
public class ChatController {

  private final ChatClient chatClient;

  @PostMapping("/chat")
  public Flux<String> chat(
      @RequestParam String message,
      @RequestParam String sessionId) {

    return chatClient
      .prompt()
      .advisors(
        new MessageChatMemoryAdvisor(memory),
        new QuestionAnswerAdvisor(vectorStore)
      )
      .user(message)
      .advisors(a -> a
        .param(CHAT_MEMORY_CONVERSATION_ID_KEY,
               sessionId))
      .stream()
      .content();
  }
}
application.yml 
spring:
  ai:
    ollama:
      base-url: http://localhost:11434
      chat:
        options:
          model: mistral
          temperature: 0.7
    vectorstore:
      pgvector:
        index-type: HNSW
        distance-type: COSINE_DISTANCE

Getting Started

Setup in 4 Steps

Get the chatbot running locally in under 5 minutes.

🦙
Install Ollama & pull a model
Download Ollama from ollama.ai, then run: ollama pull mistral or ollama pull llama3
🐋
Start PGVector (for RAG modules)
Run docker compose up -d in any module directory that uses vector search. Spins up Postgres with pgvector extension.
☕
Run the Spring Boot app
Navigate to any use-case folder and run ./gradlew bootRun. The app starts on port 8080 and connects to your local Ollama instance.
🧪
Test with HTTP or the test suite
Use curl -X POST localhost:8080/chat?message=Hello or run ./gradlew test. Each module includes integration tests.
Terminal — Quick Start 
# 1. Clone the repo
git clone https://github.com/zainkhan1994/Ollama-Chatbot.git
cd Ollama-Chatbot

# 2. Pull a model via Ollama
ollama pull mistral

# 3. Run the chatbot use-case
cd use-cases/chatbot
./gradlew bootRun

# 4. Chat!
curl -X POST "localhost:8080/chat?message=Tell+me+about+Spring+AI"
